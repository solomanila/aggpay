# 系统功能概述（面向投资人）

我们的跨境支付运营平台已完成端到端的功能规划与 API 设计，覆盖从通道配置、资金调度到风控审计的全链路能力。核心亮点如下：

## 统一运营中枢
- **仪表盘 & 监控**：Dashboard 模块提供多维度指标（成功率、延迟、容量、告警），支持银行、商户、通道视角的实时洞察与演练操作，确保全球业务连续性。
- **通知与导出**：下载管理与 Telegram/SMS/Ops 工具建立统一的通知、导出、批量任务体系，实现跨团队协作与合规留存。

## 支付与资金能力
- **收付通道管理**：Payment 套件涵盖主体、通道、限流、兜底、对账、黑名单等 10+ 主题场景，匹配复杂的多区域多通道策略诉求。
- **银行户调度**：Bank Accounts 模块管理供应商、账户、绑定、实时看板、流水、记账、映射等关键环节，支持实时限额、维护窗口与风险预警。
- **订单与结算**：Orders 模块包括收款/出款/回滚/代付队列/批量/审批/查单，配合 Finance 模块的账单、提现、代理结算、系统充值，实现“订单→账单→回款”闭环。

## 生态与运营
- **商户与代理**：Merchant/Agent 模块沉淀主体档案、标签、KPI、渠道绑定、配额、结算，帮助销售与运营团队进行精细化管理。
- **运营工具箱**：Ops 系列包含短信/OTP、短信转发设备、批量操作、VPS 管理、Telegram 通知等，增强跨地域应急调度与自动化效率。
- **系统治理**：System 模块统一管理内部账单、后台用户、角色、权限、灰度配置及审计，保证平台安全与合规。

## 技术与接口底座
- 全部模块均按照现代 Spring Cloud + RESTful 规范编写 API 需求文档，配合 pay.sql 与 pay-service 的实体/Service/Mapper 体系，便于快速落地。
- 每个前端页面对应一个详细的 API 设计（位于 `md/` 子目录），包含数据来源、接口参数、响应结构、补充表结构，确保需求闭环。

通过上述能力，平台不仅能够支撑现有跨境支付业务的稳定运营，还具备快速扩展新区域、新通道及合作方的弹性，为资本方持续输出可衡量、可复制的收益模型。***
