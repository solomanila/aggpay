## 踩坑记录与经验总结

### 1. 所有容器必须用 `--network host`

**现象**：Docker daemon 配置了 `{"iptables": false}`（因私网实例缺少 iptables 内核模块），bridge 网络下容器无法访问外部 IP，服务启动即报连接拒绝。

**结论**：本环境所有容器（包括 RocketMQ、Java 服务）均需加 `--network host`，不能用默认 bridge 或 `-p` 端口映射。

---

### 2. 私网实例无外网，镜像只能从 gateway 传入

**现象**：私网实例（10.0.1.29/167/19/38/54）无 NAT Gateway，无法访问 ECR 也无法 `dnf install`。

**结论**：所有镜像在 gateway（10.0.1.30）上 pull，再用以下命令传入各私网机器：

```bash
sudo docker save <image> | gzip | ssh -i ~/aggpay_key.pem ec2-user@<目标IP> "gunzip | sudo docker load"
```

---

### 3. RocketMQ broker 用 bridge 网络会 exit 253

**现象**：broker 容器持续 Restarting，exit code 253，无任何日志输出。

**原因**：bridge 网络在 `iptables: false` 下无法绑定端口，JVM 启动即崩溃。

**修复**：删掉旧容器，加 `--network host` 重启，同时 namesrv 也改为 `--network host`：

```bash
sudo docker stop rmq-broker && sudo docker rm rmq-broker
sudo docker run -d --name rmq-broker --network host --restart always \
  -e JAVA_OPT_EXT="-Xms256m -Xmx512m -Xmn128m" \
  apache/rocketmq:4.9.7 \
  sh mqbroker -n 10.0.1.54:9876 brokerIP1=10.0.1.54 autoCreateTopicEnable=true

sudo docker stop rmq-namesrv && sudo docker rm rmq-namesrv
sudo docker run -d --name rmq-namesrv --network host --restart always \
  apache/rocketmq:4.9.7 sh mqnamesrv
```

---

### 4. Nacos config 客户端自动启用，连接 127.0.0.1 失败

**现象**：admin-service 启动报 `Connection refused: /127.0.0.1:9848`，Nacos config 客户端用默认地址。

**原因**：`bootstrap.yml` 只配置了 `discovery.server-addr`，但 jar 包含 Nacos config 依赖，config 客户端自动启用并使用默认地址 `127.0.0.1:8848`，无视传入的 `NACOS_SERVER` 变量。

**修复**：docker run 加环境变量禁用 Nacos config（本项目配置全在本地 yml，不从 Nacos 拉取）：
```
-e SPRING_CLOUD_NACOS_CONFIG_ENABLED=false
```

---

### 5. pay-service 无外网，Telegram Bot 需通过 gateway squid 代理

**现象**：pay-service 日志持续报 `Network is unreachable`，连不上 `api.telegram.org`。

**原因**：私网实例无 NAT Gateway，无外网出口。

**修复**：在 gateway（10.0.1.30，有公网）上安装 squid 代理：

```bash
sudo dnf install -y squid
sudo sed -i '/^http_access deny all/i acl payvpc src 10.0.1.0\/24\nhttp_access allow CONNECT payvpc\nhttp_access allow payvpc' /etc/squid/squid.conf
sudo systemctl start squid && sudo systemctl enable squid
```

pay-service docker run 加 `JAVA_TOOL_OPTIONS` 环境变量走代理，同时用 `nonProxyHosts` 排除内网地址（否则 Nacos 也会走代理导致连接失败）：

```
"-e" "JAVA_TOOL_OPTIONS=-Dhttps.proxyHost=10.0.1.30 -Dhttps.proxyPort=3128 -Dhttp.proxyHost=10.0.1.30 -Dhttp.proxyPort=3128 -Dhttp.nonProxyHosts=10.0.1.*|localhost|127.* -Dhttps.nonProxyHosts=10.0.1.*|localhost|127.*"
```

---

### 6. admin-service 缺少 rocketmq 配置导致启动失败

**现象**：`connect to null failed`，`CreditOrderIncomeConsumer` 初始化时 nameserver 地址为 null。

**原因**：`admin-service/application-prod.yml` 遗漏了 `rocketmq.name-server` 配置，而 admin-service 有 RocketMQ consumer。

**修复**：在 `application-prod.yml` 补充：
```yaml
rocketmq:
  name-server: ${ROCKETMQ_NAME_SERVER:127.0.0.1:9876}
  producer:
    group: admin-service-producer-group
```
并重新 build + push 镜像。

## Nginx 配置（gateway 台）

```bash
sudo vim /etc/nginx/conf.d/payadmin.conf
```

内容：
```nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 10s;
        proxy_read_timeout 60s;
    }
}
```

> 注意：需删除 `/etc/nginx/nginx.conf` 中默认的 `server { listen 80; server_name _; ... }` 块，否则 conf.d 配置被忽略。

```bash
sudo nginx -t && sudo systemctl reload nginx
```
